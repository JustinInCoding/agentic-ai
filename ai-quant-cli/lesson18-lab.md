# Lesson 18 实操手册 · 自己造一个 AI 量化投研系统

> 目标:从一个**空目录**，让 Claude Code 造出「财报 PDF → HTML 投研报告」系统。
> 下面只给你要发给 Claude Code 的 prompt，照着一步步发即可。

## 0. 起步
- 新建一个**你自己的空目录**（如 `~/my-quant/`），进去 —— **不要在本仓库的 `ai-quant-cli/` 里做，那是做好的参考答案**。
- 建 `data/`，下载一份 A 股年报 PDF 放进去（下载方式见本仓库 `data/README.md`）。
- 启动 `claude` → 登录 → 切 **Auto 模式**。
- 想省事就把本仓库的 `BUILD_PLAN.md` 拷进来，发「读 BUILD_PLAN.md，一步步把系统做出来，每阶段停一下」让它自己跑；想一步步练，照下面来。

## 1. 先设计，别写代码
> 我要从零做一个本地量化投研系统：把 `data/` 里的年报 PDF，自动解析三张合并报表、由你做财务风险研判、出图、生成一份带图表的 HTML 投研报告。先别写代码——先给我系统设计：分层架构、各层模块与职责、层间数据契约、技术选型、目录结构、你预见的风险点。约定：分析判断由你直接做，Python 脚本只做确定性工作（解析 / 出图 / 汇编 HTML），不要在代码里调用大模型 API。

看一眼方案、提一句你想改的（例：「中间结构化数据落盘成文件，方便调试」），确认后让它搭项目骨架，再自己敲 `/init` 生成 CLAUDE.md。

## 2. 解析财报
> 写解析脚本：从年报 PDF 定位三张合并报表（资产负债表 / 利润表 / 现金流量表），把关键科目连同期末、期初抽成结构化数据落盘到 `data/`；抽完用会计恒等式（资产 = 负债 + 所有者权益）自检。先只解析、不分析，遇报错你自己修到跑通。

## 3. 风险研判 + 三表勾稽
> 基于结构化数据做隐性财务风险研判（应收 / 存货 / 盈利质量 / 资本开支 / 集中度等），每条给数据依据；再做三表勾稽交叉验证（如净利润 vs 经营现金流、营收增速 vs 应收账款增速），每条给依据。

挑一条结论追问它：
> 这条结论你是从哪几张表、哪些科目推出来的？算给我看看。

## 4. 出图
> 用结构化数据给关键指标（营收、净利润、毛利率、现金流、存货、应收等）出趋势图 / 结构图，保存成图片供 HTML 报告嵌入，遇报错你自己修。

## 5. 生成 HTML 报告
> 把结构化数据、图表、风险研判、跨表结论汇编成一个 HTML 投研报告网页：排版清晰、图表内嵌、浏览器可直接打开，输出到 `reports/`。

## 6. 一键端到端
> 把整条流程串成一个一键入口：给一句指令 + 指定 `data/` 里的年报，就自动跑完 解析 → 研判 → 出图 → 生成 HTML 报告。用现有年报端到端跑一遍。

## 做完
拿本仓库的 `ai-quant-cli/`（`src/` + `CLAUDE.md`）当参考答案，对照你和它差在哪。再换一只标的（或换一期）重跑一份，体会「造好一次、反复复用」。
